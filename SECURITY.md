# Security Policy

## Supported Versions

| Version | Supported |
|---------|----------|
| 1.x.x   | Yes      |

## Reporting a Vulnerability

If you discover a security vulnerability in the AI TRiSM Index data, tooling, or infrastructure, please report it responsibly.

### How to Report

1. **Email:** Send details to `alpha.one.hq@proton.me` with subject line `[SECURITY] ai-trism-index`
2. **GitHub:** Open a [private security advisory](https://github.com/alpha-one-index/ai-trism-index/security/advisories/new)

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 5 business days
- **Resolution target:** Within 30 days for critical issues

### Scope

- Data integrity issues (incorrect vendor data or market figures)
- CI/CD pipeline vulnerabilities
- GitHub Pages site security
- Dependency vulnerabilities

### Out of Scope

- Vulnerabilities in third-party vendors listed in the index
- Social engineering attacks
- Denial of service attacks against GitHub infrastructure

## Responsible Disclosure

We follow coordinated vulnerability disclosure. We ask that you:
- Allow reasonable time for us to address the issue before public disclosure
- Act in good faith to avoid data destruction or service disruption
- Do not access or modify other users' data
